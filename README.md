# 🚀 Multi-Model Machine Learning Portal

A high-performance Flask-based web application integrating five distinct Machine Learning models into a single, cohesive user interface. This suite demonstrates the practical application of AI across various domains, ranging from tabular data classification to advanced image and audio processing.

## 🌟 Key Features

The portal includes the following five specialized AI solutions:

1.  **🏦 Loan Approval Classification**: Predicts the likelihood of loan approval based on demographic and financial data using highly-tuned classification algorithms.
2.  **🛒 Ecommerce Customer Churn Prediction**: Analyzes customer behavior and engagement to predict potential churn, enabling proactive retention strategies.
3.  **🎭 Live Face Mask Detection**: A real-time computer vision system that detects whether individuals are wearing face masks correctly.
4.  **🍎 Fruit & Vegetable Classification**: An image recognition model capable of identifying various types of produce with high precision.
5.  **🎤 Indian Language Audio Classification**: A specialized audio processing engine that identifies major Indian languages from speech samples using Mel-spectrogram analysis.

## 🛠️ Technology Stack

- **Backend**: Python, Flask
- **Machine Learning**: Scikit-learn, XGBoost, TensorFlow, MobileNetV2
- **Data Science**: NumPy, Pandas, Librosa (Audio), PIL (Images)
- **Frontend**: HTML5, CSS3 (Modern Glassmorphism Design), JavaScript
- **Reporting**: Automated PDF report generation for analysis results.

## 🚀 Getting Started

### Prerequisites
Ensure you have Python 3.8+ installed.

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
   cd ML_PROJECT_PORTAL
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Run the application:
   ```bash
   python app.py
   ```
   Access the portal at `http://127.0.0.1:5000/`.

## 📂 Project Structure
- `app.py`: Main Flask application entry point.
- `templates/`: HTML templates for the web interface.
- `static/`: CSS and client-side assets.
- `*.ipynb`: Research and training notebooks for each model.
- `requirements.txt`: List of required Python packages.
