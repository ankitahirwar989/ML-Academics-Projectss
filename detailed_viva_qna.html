Q4. What is a training set and a test set? Why do we separate them?

  • Training set (~80%): Data the model learns from. Parameters are adjusted based on this.
  • Test set (~20%): Data the model has never seen. Used to measure real-world performance.

Why separate? If we tested on the same data we trained on, the model could simply memorise all answers and appear 100% accurate — but completely fail on new data. The test set simulates unseen real-world input.

Analogy: Studying textbook questions (training) vs. actual exam questions (testing). If the exam uses the same questions, it doesn't test genuine understanding.

Q5. What is Overfitting? What is Underfitting? How do you fix them?

            Training Error    Test Error     Problem
Underfitting:    HIGH              HIGH       Model too simple
Overfitting:     LOW               HIGH       Model too complex / memorised
Good Fit:        LOW               LOW        Learned the actual pattern

Causes of Overfitting: - Too many parameters relative to training data - Model trained too long (too many epochs) - Noisy data that the model learned as patterns

How to fix Overfitting: - More training data - Regularisation (L1/L2 penalties in XGBoost) - Dropout layers (randomly disable neurons during training) - Early Stopping (stop training when validation loss stops improving) - Cross-validation

How to fix Underfitting: - Use a more complex model - Add more features (feature engineering) - Train for more epochs (for neural networks) - Reduce regularisation

Q6. Explain the Bias-Variance Tradeoff.

  • Bias = Error from wrong assumptions in the model. High bias → underfit.
    Example: Fitting a straight line to a curved pattern.
  • Variance = Error from sensitivity to small fluctuations in training data. High variance → overfit.
    Example: A model that perfectly follows every training point, including noise.

The Tradeoff: Reducing bias increases variance and vice versa. We want both to be low — this is the "sweet spot."

In our project: Gradient Boosting and XGBoost balance this by combining many weak (high bias) trees — the ensemble reduces variance while keeping bias low.

Q7. What is Cross-Validation? Why is it better than a single train/test split?

k-Fold Cross-Validation: 1. Split data into k equal parts ("folds"), e.g., k=5 2. Train on 4 folds, validate on the remaining 1 3. Repeat 5 times, each time using a different fold as validation 4. Average the 5 accuracy results

Why better? A single split gives one accuracy number — could be lucky/unlucky depending on which data ended up where. Cross-validation uses ALL data for both training and validation (at different times), giving a more reliable accuracy estimate.

Precise Point: We used random_state=42 to ensure the same random split every time, making experiments reproducible.

Q8. What is Data Leakage? Why is it dangerous?

Data Leakage = When information from the test set accidentally influences the training process.

Example of leakage: If you scale the entire dataset first (calculating mean/std from all data), then split into train/test — the scaler has already "seen" the test data's distribution. Your model appears to perform well in testing but will fail on truly new data.

Correct approach: Fit the scaler ONLY on training data. Then transform test data using those same training parameters. python scaler.fit(X_train) # Learn mean/std from TRAIN only X_train_scaled = scaler.transform(X_train) X_test_scaled = scaler.transform(X_test) # Apply same params to TEST

Pipeline saves you: sklearn.pipeline.Pipeline automatically applies this correctly.

🔷 SECTION 2: Algorithms — Deep Dive

Q9. What is a Decision Tree? How does it split data?

A Decision Tree is a flowchart of if-else questions. At each node, it chooses the best question (feature + threshold) that most cleanly separates the classes.

Splitting Criteria:

  1. Gini Impurity: Measures the probability of misclassifying a random sample.
  2. Gini = 1 - Σ(pᵢ²) where pᵢ = fraction of samples of class i

  3. A pure node (all one class) has Gini = 0

  4. Entropy / Information Gain:

  5. Entropy = -Σ(pᵢ × log₂(pᵢ))
  6. Information Gain = Entropy(parent) - weighted_avg(Entropy(children))
  7. We pick the split with the highest Information Gain

Example: Is CreditScore > 700? ├── Yes → Is Income > 50000? │ ├── Yes → APPROVED │ └── No → REJECTED └── No → REJECTED

Weakness: A single deep tree easily overfits. That's why we use ensembles (Random Forest, Gradient Boosting).

Q10. What is Gradient Boosting? Explain step-by-step.

Gradient Boosting is an ensemble method — it builds many weak learners (shallow decision trees) sequentially, where each tree corrects the errors of the previous one.

Step-by-step algorithm: 1. Start with a naive prediction: predict the mean of y (for regression) or log-odds (for classification) 2. Calculate residuals = actual − predicted (the errors) 3. Train a small decision tree to predict those residuals 4. Update the model: new prediction = old prediction + learning_rate × tree_prediction 5. Repeat steps 2–4 for n_estimators iterations

Key parameters: - n_estimators=50 → How many trees (more = slower, potentially better) - learning_rate=0.1 → Shrinks each tree's contribution (smaller = slower but more accurate) - max_depth=3 → Depth of each weak tree (shallow = weak learner, works best for GB)

Why it's called "Gradient" Boosting?
Because it minimises a loss function by using gradient descent — each new tree moves in the direction that reduces the loss the most.

Q11. What is XGBoost? How is it different from standard Gradient Boosting?

XGBoost (Extreme Gradient Boosting) is an optimised implementation of gradient boosting with several improvements:

Feature Standard Gradient Boosting XGBoost

L1 vs L2 Regularisation in XGBoost: - L1 (alpha): Adds sum of absolute values of weights → drives some weights to exactly 0 (feature selection) - L2 (lambda): Adds sum of squared weights → shrinks all weights proportionally

Q12. What is a Convolutional Neural Network (CNN)? Why do we use it for images?

Why not a regular neural network for images?
A 224×224 RGB image has 224 × 224 × 3 = 150,528 input values. If we connected these to 1000 neurons in a regular (Dense) layer, that's 150 million parameters — impossible to train on small datasets.

CNN key idea: Instead of connecting every pixel to every neuron, use local filters that slide over the image and detect local patterns.

Layers explained:

  1. Conv2D(filters=32, kernel_size=3×3):
  2. A 3×3 filter slides across the image
  3. Calculates dot product at each position → creates a "feature map"

  4. Early layers detect edges, later layers detect shapes/faces

  5. MaxPooling2D(pool_size=2×2):

  6. Takes the maximum value in each 2×2 region → shrinks feature maps by 75%

  7. Reduces computation, adds spatial invariance (slight shifts don't matter)

  8. GlobalAveragePooling2D:

  9. Averages each entire feature map into one number

  10. Converts 3D feature tensor → 1D feature vector

  11. Dense(256, activation='relu'):

  12. Fully connected layer — combines all features for final decision

  13. Dense(num_classes, activation='softmax'):

  14. Final output — probabilities for each class

Q13. What is ReLU? Why is it preferred over Sigmoid for hidden layers?

ReLU (Rectified Linear Unit): f(x) = max(0, x) Output: 0 if x < 0, else x (passes through positive values unchanged)

Why better than Sigmoid for hidden layers?

Problem with Sigmoid How ReLU Fixes It
Vanishing gradients: output saturates near 0 and 1, gradient ≈ 0 → deep networks stop learning ReLU gradient = 1 for all positive x → gradients flow freely
Computationally expensive (exponential) Computationally trivial (just a max operation)
Outputs not zero-centered Not zero-centered but still better in practice

Where Sigmoid IS used: Binary output layers (predict probability 0–1 for binary classification).

Q14. What is Softmax activation? How does it work?

Softmax converts a vector of raw scores (logits) into a probability distribution that sums to 1.

Formula: softmax(xᵢ) = e^xᵢ / Σ(e^xⱼ) for all j in all classes

Example: Raw logits: [2.1, 0.3, 1.5] e^2.1 = 8.17, e^0.3 = 1.35, e^1.5 = 4.48 Sum = 14.00 Softmax: [8.17/14, 1.35/14, 4.48/14] = [0.584, 0.096, 0.320] Predicted class = 0 (highest probability: 58.4%)

Why exponents? It amplifies differences — the highest score gets disproportionately more probability. Also ensures all outputs are positive.

Used in: The final Dense layer of all our multi-class models (Fruit/Veg: 36 classes, Audio: 10 classes).

Q15. What is Transfer Learning? Why did we use MobileNetV2?

Transfer Learning = Taking a model already trained on a large dataset and reusing its learned features for a new (smaller) task.

Analogy: A surgeon learning to use a new scalpel doesn't need to re-learn anatomy from scratch. Similarly, MobileNetV2 already "knows" how to detect edges, textures, shapes — we only teach it which of our categories match those shapes.

Our approach — 2 phases: 1. Phase 1 (Frozen base): Freeze all MobileNetV2 layers. Train ONLY our new classification head (Dense layers). Fast training, no risk of destroying ImageNet knowledge. 2. Phase 2 (Fine-tuning): Unfreeze the LAST FEW layers of MobileNetV2. Train with a very small learning rate. The model's feature detectors slightly adapt to our specific domain.

Why MobileNetV2 specifically? - Designed for efficiency — uses Depthwise Separable Convolutions which reduce FLOP count by ~8-9x vs standard convolutions - Pre-trained on ImageNet (1.4 million images, 1000 classes) - Input: 224×224×3 (perfect for our fruit/veg and audio models) - Much less GPU needed than VGG16/ResNet50

Q16. What are Depthwise Separable Convolutions? (Why MobileNet is efficient)

Standard Convolution: One filter processes all input channels simultaneously.
Cost: D_K × D_K × M × N × D_F × D_F operations (M input channels, N output channels, D_K kernel size, D_F feature map size)

Depthwise Separable Convolution: Split into two steps: 1. Depthwise Conv: One filter per input channel (processes channels independently) 2. Pointwise Conv (1×1): Combine results across channels

Result: ~8-9x fewer computations with similar accuracy. This is why MobileNet runs on phones.

🔷 SECTION 3: Preprocessing & Data Processing

Q17. What is StandardScaler? What does it do mathematically?

StandardScaler transforms each feature to have mean = 0 and standard deviation = 1 (Z-score normalisation).

Formula: z = (x - μ) / σ Where μ = mean of that feature in training data, σ = standard deviation

Example: Income column: [30000, 50000, 70000, 100000] μ = 62500, σ = 29155 Scaled: [-1.12, -0.43, 0.26, 1.29] ← now all roughly between -2 and +2

Why needed? Gradient-based algorithms and distance-based algorithms (KNN, SVM, Neural Networks) are sensitive to feature scale. Without scaling, income (50,000) dominates age (30) in distance calculations, even though both may be equally important.

Important: scaler.fit() on TRAIN only. scaler.transform() on both train and test.

Q19. What is a Mel-Spectrogram and how does audio classification work?

Step 1: Load audio ```python y, sr = librosa.load('audio.wav', sr=22050)

```

Why Mel scale?
Human hearing perceives pitch logarithmically — the difference between 100 Hz and 200 Hz sounds the same as between 1000 Hz and 2000 Hz. Mel scale assigns equal perceptual spacing, making it ideal for speech/language analysis.

Q21. What is the difference between .fit(), .transform(), and .fit_transform()?

Method What it does When to use
.fit(X) Learns parameters from X (e.g., mean, std for scaler; categories for encoder) On training data only
.transform(X) Applies already-learned parameters to X On any data (train or test)
.fit_transform(X) Fits then transforms in one step ONLY on training data

Rule: Never call .fit() or .fit_transform() on test data. This would cause data leakage (the scaler would learn from test statistics and give unfair results).

```python

CORRECT ✅

WRONG ❌ — leakage

A Pipeline chains multiple processing steps into one object. Calling pipeline.fit(X_train) applies each step in order, and pipeline.predict(X) processes new data through all steps automatically.

Q27. Define Accuracy, Precision, Recall, and F1-Score. When does each matter?

Given a confusion matrix: Predicted Positive Predicted Negative Actual Positive: TP FN Actual Negative: FP TN

Metric Formula Real meaning
Accuracy (TP+TN)/(TP+TN+FP+FN) % predictions correct overall
Precision TP/(TP+FP) Of all I flagged as positive, how many actually were?
Recall TP/(TP+FN) Of all actual positives, how many did I catch?
F1-Score 2×(P×R)/(P+R) Harmonic mean — useful when classes are imbalanced

When each matters: - Medical diagnosis (cancer): High Recall is critical — missing an actual case (FN) is catastrophic - Email spam filter: High Precision is critical — flagging real emails as spam (FP) annoys users - Loan default prediction: High Recall — missing an actual defaulter costs the bank money - Imbalanced dataset: Never use accuracy alone — a model that always says "no default" can be 95% accurate on a 5% default dataset, but is completely useless

Q28. What is AUC-ROC? What does it tell you?

ROC Curve plots True Positive Rate (Recall) vs False Positive Rate at every possible decision threshold.

AUC (Area Under Curve): - AUC = 1.0 → Perfect model - AUC = 0.5 → Random guessing (diagonal line) - AUC = 0.0 → Perfectly wrong

Why useful: Accuracy is threshold-dependent (usually 0.5). AUC measures performance across ALL thresholds — shows how well the model discriminates between classes regardless of threshold choice.

In our project: Loan model can output predict_proba() → probability of approval. AUC tells us how well it separates approvals from rejections.

Q34. What is Dropout? How does it prevent overfitting?

Dropout(rate=0.5) randomly sets 50% of neuron outputs to 0 during each training step.

Why this prevents overfitting: - Neurons can't co-adapt (rely on each other) - Forces each neuron to independently learn useful features - Acts like training an ensemble of many different sub-networks

At test time: Dropout is automatically turned off — all neurons active. The outputs are scaled by (1 - dropout_rate) to maintain consistent expected values.

Important: Dropout is only active during model.fit(). During model.predict() or model.evaluate(), it's disabled automatically.

Q36. What does base.trainable = False do? When do you set it to True?

Q37. What is the vanishing gradient problem? How does ReLU help?

Vanishing Gradient: In deep networks with Sigmoid or Tanh activations, gradients become very small (near 0) as they flow backwards through many layers. Early layers receive nearly no gradient → they can't learn → deep networks fail.

Why? Sigmoid's derivative maxes out at 0.25. Multiplying 0.25 × 0.25 × 0.25 across 10 layers = 0.000001 — effectively zero.

ReLU fix: For positive inputs, the gradient of ReLU is exactly 1.0. Multiplying 1×1×1×1 = 1 → gradient flows through without shrinking.

Dying ReLU problem: If inputs to ReLU are always negative, it outputs 0, has gradient 0, and that neuron never updates ("dies"). Fixed by: Leaky ReLU (small negative slope) or ELU.

Q38. Why do we use n_mels=128 in the Mel-Spectrogram?

n_mels=128 means we divide the frequency range into 128 Mel-scaled frequency bins.

More bins = more frequency resolution = more detail = larger array. The resulting spectrogram has shape (128, time_frames).

We then resize this to (224, 224, 3) for MobileNetV2. 128 is a common choice — large enough to capture phonetic detail, small enough for efficient processing. Powers of 2 (64, 128, 256) are conventional in audio processing.

Q39. What is the learning rate? What happens if it's too high or too low?

The learning rate controls how much the model's weights are updated in response to the gradient.

new_weight = old_weight - learning_rate × gradient

Learning Rate Effect
Too HIGH Weights oscillate wildly, loss increases, training diverges
Too LOW Training is very slow, may get stuck in local minima
Just right Steady convergence to a good solution

In Gradient Boosting: Learning rate = shrinkage factor for each tree's contribution. Lower LR + more estimators = better generalisation but slower.

In fine-tuning (Transfer Learning): We use 1e-5 (very small) to avoid catastrophically overwriting pre-trained features (called "catastrophic forgetting").

Q40. What is an Epoch vs a Batch vs an Iteration?

Term Definition
Epoch One complete pass through the entire training dataset
Batch A small subset of training data processed together before updating weights
Iteration One forward + backward pass for one batch

Relationship: If dataset = 1000 samples, batch_size = 32: Iterations per epoch = ceil(1000 / 32) = 32 iterations If training for 20 epochs → 20 × 32 = 640 total iterations

Why mini-batches? Processing the full dataset at once (batch gradient descent) requires too much memory. Processing one sample at a time (stochastic GD) is noisy. Mini-batches (32–256) balance memory and noise.

Q41. What would happen if you didn't normalise images before feeding to MobileNetV2?

MobileNetV2's convolutional filters have weights learned assuming inputs in the [-1, 1] range. If you feed raw [0, 255] values: 1. Activations would be 255× larger than expected 2. ReLU would saturate 3. The model would output meaningless predictions Normalization is not optional — it must match exactly what was used during ImageNet pre-training.

Q42. Why do ensemble methods like Gradient Boosting outperform single decision trees?

A single deep decision tree has high variance — it overfits every noise pattern in training data. Different training samples would produce completely different trees.

Ensemble insight (Bias-Variance decomposition): - Many weak learners (high bias, low variance) combined → reduce bias through boosting - Random perturbations (different data subsets, features) → reduce variance through averaging

Boosting specifically reduces bias — each tree corrects systematic errors of the previous ensemble. Error approaches 0 with enough trees (as long as each tree is better than random).

Q43. What is One-Hot Encoding's "dummy variable trap"? How do you avoid it?

If a feature has k categories, OHE creates k binary columns. These k columns are perfectly multicollinear — knowing the first k-1 tells you the k-th.

For linear models (Logistic Regression), this causes numerical instability. Fix: Use drop='first' in OneHotEncoder to create only k-1 columns.

For tree-based models (Decision Tree, XGBoost, Gradient Boosting): This doesn't cause problems — trees don't solve linear equations, so multicollinearity isn't an issue. We generally don't need to drop the first column.

Q44. What is the difference between model.predict() and model.predict_classes()?

predict_classes() was a Keras convenience method that returned the class index directly (argmax).

It was removed in TensorFlow 2.6+. The correct modern approach: python probs = model.predict(x) # Returns probabilities [0.1, 0.8, 0.1] class_idx = np.argmax(probs, axis=1) # Returns [1] — index of highest prob

Q45. If your model has 99% accuracy but is useless, what's wrong?

Class imbalance problem.

Example: 99% of loan applications are approved, 1% default. A model that always predicts "Approved" achieves 99% accuracy — but never catches a single defaulter.

Solution: - Use Recall, Precision, F1-Score, AUC-ROC instead of accuracy - Oversample minority class (SMOTE) - Undersample majority class - Use class_weight='balanced' in sklearn models - Adjust decision threshold

Q47. What is top_db=20 in librosa.effects.trim()?

Sounds below 20 dB relative to the peak amplitude are considered silence and removed from the beginning and end of the audio clip.

Why trim? Indian language audio clips might have leading/trailing silence from recording. This silence adds no language information but wastes processing. Trimming ensures the model focuses on actual speech content.

20 dB is a practical threshold — quiet background noise is typically under 20 dB below peak, while actual speech is above.

Q49. What is the difference between Keras Sequential and Model (Functional API)?

Sequential API Functional API
Structure Linear stack of layers Any DAG (directed acyclic graph)
Use case Simple models Multi-input, skip connections, shared layers
Usage model = Sequential([layer1, layer2]) x = layer(input); model = Model(inputs, x)

We used Functional API for Transfer Learning: python base = MobileNetV2(include_top=False, input_shape=(224,224,3)) x = GlobalAveragePooling2D()(base.output) x = Dense(256, activation='relu')(x) output = Dense(num_classes, activation='softmax')(x) model = Model(inputs=base.input, outputs=output) This is necessary because MobileNetV2 is itself a complex non-sequential model.

Q50. What preprocessing do you do before feeding an image to the face mask model?

```python from PIL import Image import numpy as np

img = Image.open(uploaded_file) # Open file img = img.convert('RGB') # Ensure 3 channels (not RGBA/greyscale) img = img.resize((128, 128)) # Face mask model uses 128×128 arr = np.array(img) / 255.0 # Normalize to [0, 1] arr = np.expand_dims(arr, axis=0) # Shape: (1, 128, 128, 3) — batch dimension prediction = model.predict(arr) # Returns [[mask_prob, no_mask_prob, incorrect_prob]] ```

Why expand_dims? Keras models always expect a batch dimension. A single image has shape (128, 128, 3) but the model expects (batch_size, 128, 128, 3). expand_dims(arr, axis=0) adds the batch dimension → (1, 128, 128, 3).
